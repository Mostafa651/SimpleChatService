Core Features

üí¨ Real-time Messaging

The chat system provides instant message delivery using our optimized signal-based communication system built on GoodSignal for maximum performance.

Basic Message Sending

-- Client-side message sending
local ChatEvents = require(ReplicatedStorage.ChatEvents)

-- Send a message
ChatEvents.SendMessage:Fire("Hello, world!")

-- Listen for incoming messages
ChatEvents.MessageReceived:Connect(function(playerName, message, timestamp)
    print(playerName .. ": " .. message)
end)

Specifications

  • Max Message Length: 200 characters (configurable)
  • Message Rate: 5 messages per 10 seconds (configurable)
  • History Limit: 100 messages (configurable)
  • Latency: < 50ms average delivery time

Admin System

Comprehensive admin tools for server moderation with role-based permissions and extensive logging capabilities.

Available Commands

/mute Admin

Temporarily or permanently mute a player

Usage: /mute [player] [duration] [reason]
Examples:
  • /mute Player1 5m Spamming
  • /mute Player2 1h Inappropriate language
  • /mute Player3 permanent Harassment
/unmute Admin

Remove mute from a player

Usage: /unmute [player]
/kick Admin

Remove a player from the server

Usage: /kick [player] [reason]
/ban Super Admin

Permanently ban a player

Usage: /ban [player] [reason]
/announce Moderator

Send server-wide announcement

Usage: /announce [message]
/clear Moderator

Clear chat history for all players

Usage: /clear

Permission Levels

Role Level Permissions Commands
Player 0 Send messages, use emojis None
Moderator 1 Mute players, clear chat /mute, /unmute, /clear, /announce
Admin 2 Kick players, advanced moderation All moderator + /kick
Super Admin 3 Ban players, system configuration All admin + /ban, /config

Events & Signals

The chat system uses a comprehensive event system built on GoodSignal for reliable and performant communication between client and server.

üì§ Client Events

üì• Server Events
